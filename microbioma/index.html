MicroBioma Browser Extension

Privacy Policy

This Privacy Policy explains how MicroBioma handles information when you install and use the extension. MicroBioma is a compact, local micro-ecosystem simulation that lets users seed, observe, and reset a virtual petri-dish environment.

Effective date: July 14, 2026 Last updated: July 14, 2026
Privacy summary MicroBioma does not collect personal information, does not track browsing activity, does not display advertising, and does not send simulation activity to external servers. The extension stores only limited user preferences locally in the browser.

1. Scope of This Policy

This Privacy Policy applies to the MicroBioma browser extension and to the features provided through its extension interface. It explains what information the extension processes, why that processing is necessary, where information is stored, and what categories of information are not collected.

MicroBioma is intended to function as a lightweight visual sandbox. Users can add seeds, watch simple organisms spread and interact, adjust available visual or motion settings, and reset the simulation to create a new balance. These actions are part of the local simulation experience and are not used to create a user profile.

This policy does not govern the privacy practices of the browser, the browser-extension marketplace, the operating system, or websites that may be open while the extension is installed. Those services operate under their own terms and privacy policies.

2. How MicroBioma Works

MicroBioma runs inside the browser as a self-contained extension. When a user opens or activates the extension, the interface displays a simulated petri-dish environment. The user may place seeds, observe growth, change supported visual settings, and restart the environment.

The simulation logic runs locally in the browser. The extension does not need to send simulation events, visual state, or interaction history to a developer-operated server in order to function.

MicroBioma does not require account registration, login credentials, a subscription, or cloud synchronization. The extension is designed so that its primary features remain available without creating an identity or remote account for the user.

3. Information MicroBioma Does Not Collect

MicroBioma does not collect, transmit, or maintain personal information. The extension is not designed to identify users or to associate simulation activity with a real person.

In particular, MicroBioma does not collect:

  • Names, email addresses, telephone numbers, postal addresses, or account details;
  • Passwords, authentication tokens, payment information, or financial details;
  • Browsing history, visited URLs, bookmarks, search queries, or open-tab activity;
  • Website content, form entries, private messages, or files from the user's device;
  • IP addresses, precise location, device identifiers, or demographic information;
  • Advertising identifiers, behavioral profiles, telemetry, or analytics data;
  • Simulation histories, seed-placement histories, organism activity logs, or reset history.

Because these categories of information are not collected, the developer does not maintain a database of identifiable users or a remote archive of MicroBioma activity.

4. Local Processing of Simulation Activity

Actions performed inside MicroBioma are processed locally for the immediate purpose of updating the simulation. For example, when a user drops a seed, the extension uses that input to place a starting element in the simulated environment. When organisms spread or interact, the extension updates the visible state of the petri-dish view.

These interactions are not treated as personal information. They are not uploaded, analyzed for advertising, used to infer user characteristics, or combined with activity from other websites or applications.

MicroBioma does not create a persistent remote profile based on the user's preferred seed placement, simulation style, motion level, or reset behavior.

5. Local Storage and User Preferences

MicroBioma uses the browser's storage capability only to remember limited settings selected by the user. Depending on the installed version, these settings may include whether the extension is enabled, the selected style, motion level, display preferences, or other configuration options directly related to the simulation.

Why the storage permission is required The storage permission allows MicroBioma to restore the user's selected preferences across browser sessions. It is not used to collect browsing history, store website content, or create a remote user profile.

Stored preferences remain in the browser's extension storage. They are not sent to the developer, sold to third parties, or used for marketing.

Users can remove locally stored preferences by resetting the extension, clearing browser extension data, or uninstalling MicroBioma.

6. Browser Permissions

MicroBioma requests only the browser permission needed to preserve extension settings. The extension does not request broad access to browsing history, bookmarks, passwords, downloads, location, or website content for its declared functionality.

storage

The storage permission is used only to save user preferences locally. This may include enabled state, style selection, motion level, or related display settings. The permission is not used to transmit information to an external service.

If a future version requires additional permissions, this Privacy Policy will be updated to explain their purpose before those permissions are represented as part of MicroBioma.

7. Advertising, Analytics, and Tracking

MicroBioma does not display advertisements and does not contain advertising SDKs, sponsored content systems, affiliate tracking, or monetization code based on user data.

The extension does not use analytics platforms, tracking pixels, browser fingerprinting, session replay tools, cross-site identifiers, or behavioral profiling.

MicroBioma does not follow users across websites, create interest profiles, or share information with advertisers, analytics providers, marketing platforms, or data brokers.

8. Remote Code and External Services

MicroBioma does not use remote code. The executable logic and assets required for the extension's simulation are included in the installed extension package.

The extension does not download additional executable scripts from an external server at runtime and does not depend on third-party processing services to run the simulation.

The browser or browser-extension marketplace may independently process information related to installation, updates, security, or store usage. Those practices are outside MicroBioma's control and are governed by the relevant platform's own policies.

9. Data Sharing, Sale, and Disclosure

MicroBioma does not sell, rent, license, trade, or disclose personal information. The extension does not share simulation activity or locally stored preferences with advertisers, analytics companies, data brokers, social networks, or marketing services.

Because MicroBioma does not collect personal information, there is ordinarily no user data available for commercial sharing or disclosure.

The developer does not operate a user-data marketplace, advertising exchange, or third-party profiling service connected to MicroBioma.

10. Data Security and Retention

MicroBioma reduces privacy risk by avoiding the collection and transmission of personal information. Simulation activity is processed locally, and the only retained information consists of limited user preferences stored in the browser.

Locally stored preferences remain available until they are changed by the user, cleared through browser settings, removed by resetting the extension, or deleted when the extension is uninstalled.

No technical system can eliminate every possible security risk. However, MicroBioma follows a limited-purpose design and does not maintain a developer-controlled archive of simulation activity or identifiable user information.

11. Children's Privacy

MicroBioma does not knowingly collect personal information from children or from users of any age. The extension does not provide account registration, messaging, social networking, targeted advertising, or features intended to gather information from minors.

Because the extension does not collect personal information, it does not create child profiles or maintain age-based user records.

12. Changes to This Privacy Policy

This Privacy Policy may be revised if MicroBioma's features, permissions, or information practices change. Any updated version will include a revised effective date and will describe the extension's current behavior.

If a future version introduces new data collection, cloud synchronization, analytics, external processing, or additional browser permissions, this policy will be updated accordingly before those practices are represented as part of MicroBioma.
